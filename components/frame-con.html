功能组件

全部组件分类

UIF提供了超过 30 个模块化并可扩展的组件，它们可以自由结合。组件根据自己的用途及功能分为不同的类型。

默认

这些组件通常用来实现HTML元素的跨浏览器标准化功能，并添加了一些非常基础的样式。

布局

充分运用我们完全响应式的流体网格系统和面板，常见的布局组成部分如博客文章和评论以及其他一些实用的效果类型。

导航

UIF 提供了不同形式的导航组件，如导航栏和侧导航。使用面包屑或通过一个分页来翻阅文章。

元素

基本的HTML元素样式，如表格和表单。这些组件使用自有的类。它们不会因想到任何默认的元素样式。

常用

在这里你可以找到一些经常在内容中使用的组件，如按钮、图标、徽章、缩略图、进度条等等。

JavaScript

这些组件主要依赖于 JavaScript 的淡出显示与隐藏内容，比如下拉菜单，模态对话框，弹出式画布和提示组件等。

多平台适配

UIF 包含一系列为各种不同的浏览器宽度实现响应式内容的类。下面的表格提供了一个关于一些可用的CLASS名称以及相应的设备概述。

屏幕宽度 class名称 设备
Mini 小于 479px 手机纵向
Small 480px 到 767px 手机横向
Medium 768px 到 959px 平板电脑纵向
Large 960px 到 1199px 台式机或平板电脑横向
Xlarge 1200px 或以上 大屏幕设备

CSS 命名规律

为了提高识别度，所有的UIF类均添加不同前缀以命名区分。由于历史原因，部分不规则命名仍存在，后期逐步去掉~

前缀 概述
g- 基础性的框架布局，主要应用于项目整体的布局规律
s- 主要应用于项目外观样式
f- 功能性的样式，比如清除浮动，对齐等
m- 局部区域小模块的命名和区分

JavaScript 使用

你可以通过为HTML元素添加 data-uk-* 属性来使用前端框架中存在的JS功能

html代码绑定

<button data-uk-button>My Button</button>

当然你也可以通过 jQuery API 来使用这些组件。

JS代码绑定

$(".button").uk("button");
